A differential privacy census dashboard

Census under noise

How much does privacy cost? Real 1994 US census data, 45,222 records after cleaning. Move the privacy dial and watch true statistics get obscured by calibrated noise, the mathematical price of protecting any one person's record.

What you're looking at

  1. Two numbers, one true, one fuzzed. Every query below computes the real answer from 45,222 real people, then adds random mathematical noise before showing it to you. That's how differential privacy publishes statistics without exposing anyone's individual record.
  2. The epsilon dial controls how much noise. Drag it left for strong privacy (more noise, less accurate), right for weak privacy (less noise, more accurate). This is the core tradeoff the whole field is built around.
  3. The noisy number changes every time on purpose. Switching tabs, changing the query, or clicking ↻ Draw new noise sample all draw a fresh random value. If it looked the same every time, it wouldn't be private.
  4. "Expected average error" is different from "this draw." One is a single random outcome; the other is what the math predicts you'd see on average across many draws. Only the epsilon, mechanism, or query changes the second one.

Expected error vs. privacy budget

The fundamental tradeoff: lower ε means stronger privacy but noisier answers. Line = analytic expectation, dots = 100-draw simulation at each ε. The marker shows your current dial position.

Both axes are log-scaled: the tradeoff compounds multiplicatively, not by fixed amounts, so small drags near either end of the epsilon dial matter far more than they look like they should.

How the noise is actually computed ↓

Laplace mechanism. For a query f with global sensitivity Δf (the most one person's record can change the true answer), release f(D) + Lap(Δf/ε). Sampled here as -b·sign(u)·ln(1-2|u|) for u ~ Uniform(-0.5, 0.5) and scale b = Δf/ε. This satisfies pure ε-differential privacy.

Gaussian mechanism. Release f(D) + N(0, σ²) with σ = Δf·√(2·ln(1.25/δ)) / ε, satisfying (ε,δ)-differential privacy: it leaks with probability δ, which is why δ should be far smaller than 1/n. This demo uses the standard analytic-Gaussian approximation, valid for ε < 1; treat larger ε values here as illustrative only.

Sensitivity per query type. Counting queries (how many high earners, how many in bin X) have Δ = 1: adding or removing one person changes a count by at most one. Bounded mean queries (mean age, mean hours) use Δ = (max−min)/n, treating the record count n as public.

Composition caveat. Every tab here treats its query as the only thing ever released from this dataset. A real system spends a finite total privacy budget across every query it will ever answer. Ask ten questions at ε = 1 each and you've spent ε = 10, not ε = 1. The histogram tab is the one exception: because each record lands in exactly one bin, parallel composition means noising every bin independently at scale 1/ε still only costs ε total, regardless of how many bins there are.
